Global Static Website

A globally distributed static website built on AWS that leverages a Content Delivery Network (CDN) for sub-second latency and automated CI/CD for zero-touch deployments. This architecture ensures high availability through geographic data redundancy.

Project Objective

To design a high-performance web hosting solution that utilizes edge caching for global speed, enforces mandatory HTTPS for security, and implements cross-region replication to ensure 99.99% data durability.

Architecture Flow

GitHub (Source) → AWS CodePipeline (CI/CD) → Amazon S3 (Origin) → Amazon CloudFront (Global CDN)
  • Automated Deployment: Every Git push triggers a pipeline that automatically updates the primary S3 bucket.
  • Edge Acceleration: CloudFront caches content at 400+ locations worldwide, serving users from the node closest to them.
  • Secure Delivery: SSL/TLS termination at the edge ensures all traffic is encrypted via HTTPS using AWS Certificate Manager.
  • Geographic Redundancy: S3 Cross-Region Replication (CRR) maintains a live, synchronized backup in a secondary AWS region.

Technical Highlights

  • SSL Termination: Offloaded encryption overhead to the CDN edge, reducing handshake latency for global visitors.
  • Cache Invalidation: Configured automatic cache purging to ensure users receive the latest site updates immediately after deployment.
  • Disaster Recovery: Engineered a multi-region architecture to protect against regional AWS outages.
  • CI/CD Optimization: Implemented automatic artifact extraction to streamline the flow from GitHub to S3 storage.
